## 7. DATA FLOW — EXECUTION FROM TAP TO SAVE

```
1. User taps "Open PDF"
   ↓
2. Capacitor Filesystem plugin opens Android file picker
   ↓
3. PDF bytes read into ArrayBuffer
   ↓
4. pdfjs loads ArrayBuffer → renders page 1 to canvas
   ↓
5. pdf-lib loads same ArrayBuffer → held in memory for editing
   ↓
6. Konva canvas overlaid on top of pdfjs canvas (same dimensions)
   ↓
7. User picks "Text" tool → taps on page
   ↓
8. A <div contenteditable> appears at tap coordinates
9. User types, picks font/size/color
   ↓
10. On "Done" → text parameters stored in annotationState[]
    ↓
11. On "Save" →
    a. pdf-lib loads current PDF bytes
    b. For each item in annotationState:
       - Text → page.drawText() with stored params
       - Drawing → convert Konva SVG paths → pdf-lib lines
       - Image → embedPng/Jpg → page.drawImage()
       - Highlight → page.drawRectangle() with opacity
    c. pdfDoc.save() → new Uint8Array
    d. Capacitor Filesystem writes bytes to device storage
    e. Success toast → green checkmark animation
```

## 10. MAINTENANCE-FREE ARCHITECTURE

### Why It Needs Zero Maintenance

```
pdf-lib      → Pure JS, no external API, no deprecation risk
pdfjs-dist   → Mozilla project, 10+ year track record
Capacitor    → Stable, major version every 2 years
Next.js      → Static export, no server to maintain
No backend   → Nothing to go down, no server bills
No auth      → No user accounts to manage
No database  → No migrations, no schema changes
No API keys  → Nothing to expire or rotate
```

### Lock Dependency Versions
```json
// package.json — pin exact versions
{
  "dependencies": {
    "pdf-lib": "1.17.1",
    "pdfjs-dist": "4.2.67",
    "next": "14.2.3",
    "@capacitor/core": "6.1.0"
  }
}
```

Pin versions. Never run `npm update` blindly. The app works on Day 1 and Day 1000 identically.

### What Could Break in 5 Years
```
Risk: Android drops WebView support → Probability: Near zero (it's core Android)
Risk: pdf-lib abandoned → Probability: Low, fork exists, pure JS
Risk: pdfjs abandoned → Probability: Zero (Mozilla maintains it)
Risk: Play Store policy change → Update APK target API (1 day work, once every 2 years)
```

The only real maintenance: **target API level updates** when Google mandates it.
That's ~4 hours of work every 18-24 months. That's it.
